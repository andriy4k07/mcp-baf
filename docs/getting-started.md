# Швидкий старт

[← Назад до README](../README.md)

Підключаємо AI-асистента до вашої бази 1С (BAF) у режимі читання: метадані,
дані через запити, повнотекстовий пошук по коду конфігурації.

## Що це і навіщо

mcp-baf — це MCP-сервер, який дає AI-асистенту доступ **на читання** до вашої
бази 1С:Enterprise (BAF: Малий Бізнес/1С:УНФ) через HTTP-сервіс. AI бачить
реальні метадані вашої конфігурації (довідники, документи, регістри, форми),
може виконувати запити (тільки `SELECT`/`ВЫБРАТЬ`), шукати по коду BSL і
читати журнал реєстрації. Нічого не пише в базу — це чиста читаюча сторона;
для створення об'єктів існує окремий проєкт `baf-write-mcp`.

### Що таке MCP

MCP (Model Context Protocol) — відкритий протокол, який дозволяє
AI-асистентам звертатися до зовнішніх інструментів і джерел даних. Його
підтримують Claude Desktop, Claude Code, Cursor, Windsurf, VS Code та інші —
повний список у [`compatibility.md`](compatibility.md). mcp-baf не прив'язаний
до конкретної нейромережі: підключається до будь-якого MCP-сумісного клієнта,
включно з локальними моделями (Ollama, LM Studio).

### Як це працює

```
┌─────────────────┐         ┌──────────────────┐        ┌─────────────────┐
│  AI-асистент    │  stdio  │     mcp-baf      │  HTTP  │  База 1С (BAF)  │
│  (будь-який     │ ──────► │  метадані,       │ ─────► │  розширення     │
│   MCP-клієнт)   │ ◄────── │  запити, пошук   │ ◄───── │  MCP_HTTPService│
└─────────────────┘         └──────────────────┘        └─────────────────┘
```

Сервер — локальний процес поруч з AI-клієнтом. Дані бази йому потрібні тільки
через HTTP-сервіс `/hs/mcp-baf`, який реєструє розширення в базі 1С.

## Вимоги

- **BAF: Малий бізнес / 1С:УНФ** на платформі **1С:Підприємство 8.3.10+**
  (мінімальна підтримувана версія; 8.5.x також підтримується).
- **Python 3.11+** на машині, де працює AI-клієнт.
- **AI-клієнт із підтримкою MCP** — детальний перелік і обмеження в
  [`compatibility.md`](compatibility.md).
- **Windows або Linux** для HTTP-сервісу 1С — на macOS HTTP-сервіси 1С не
  підтримуються (сам пакет mcp-baf на macOS працює, включно з `--install`).
- **Опційно:** локальне вивантаження конфігурації (`DumpConfigToFiles`) —
  лише якщо потрібен `search_code`.

## Встановлення

### Крок 1. Встановити пакет

```sh
git clone https://github.com/andriy4k07/mcp-baf
cd mcp-baf
python -m venv .venv
.venv\Scripts\pip install -e .      # Windows
# .venv/bin/pip install -e .        # Linux / macOS
```

Після цього в `.venv\Scripts\` (Windows) / `.venv/bin/` (Linux/macOS)
з'явиться виконуваний файл `mcp-baf` — його повний шлях знадобиться на
кроці 4.

### Крок 2. Встановити розширення в базу 1С

Розширення `MCP_HTTPService` додає в базу HTTP-сервіс на шляху `/hs/mcp-baf`
(значення захардкоджене в `RootURL` розширення — саме тому `--base` завжди
має закінчуватися на `/hs/mcp-baf`). Розширення **не змінює** вашу
конфігурацію.

> **Тільки власник бази ставить розширення на живу базу.** Якщо ви — не
> адміністратор цієї бази 1С, координуйте установку з ним.

**Автоматично (рекомендовано):**

```sh
# файлова база
mcp-baf --install "C:\шлях\до\бази"

# macOS / Linux — шлях до каталогу файлової бази
mcp-baf --install ~/Documents/InfoBase

# клієнт-серверна база (MS SQL / PostgreSQL) — формат "сервер\база" + --server
mcp-baf --install "srv-1c\buh_prod" --server --db-user Адміністратор --db-password ***

# російські синоніми об'єктів розширення (за замовчуванням — українські, --lang ua)
mcp-baf --install "C:\шлях\до\бази" --lang ru
```

Програма сама знайде встановлену платформу 1С у стандартних каталогах,
завантажить розширення через DESIGNER (`/LoadConfigFromFiles`) і оновить
конфігурацію бази (`/UpdateDBCfg`). За замовчуванням ставляться українські
синоніми (`--lang ua` — мова за замовчуванням, синоніми та мова інтерфейсу
локалізуються під `uk`).

> Якщо платформа встановлена в нестандартну папку — вкажіть шлях явно:
> `--platform "шлях/до/1cv8"`. Якщо версію платформи не видно зі шляху —
> додайте `--platform-version 8.3.13`.

> **Мінімальна підтримувана версія платформи — 8.3.10.** На старіших
> установка завершиться помилкою ще до звернення до DESIGNER.

> Перед установкою закрийте всі сеанси 1С (Конфігуратор, Підприємство) для
> цієї бази — інакше DESIGNER не зможе оновити конфігурацію.

**Вручну (якщо автоматична не спрацювала):** відкрийте базу в Конфігураторі →
**Конфігурація → Розширення конфігурації → Додати**, завантажте розширення з
`src/mcp_baf/extension_src/` і натисніть **F7** (оновити конфігурацію БД).

### Крок 3. Опублікувати HTTP-сервіс

#### Спосіб 1. Публікація через Apache або IIS (рекомендовано)

Стандартний спосіб, працює на **Windows** (IIS/Apache) і **Linux** (Apache
або ibsrv).

1. Відкрийте базу в **Конфігураторі від адміністратора**.
2. **Адміністрування → Публікація на веб-сервері**.
3. Вкажіть ім'я (наприклад `base`), відмітьте HTTP-сервіс розширення.
4. Натисніть **Опублікувати**.

URL у цьому разі: `http://localhost/base/hs/mcp-baf`.

#### Спосіб 2. Вбудований HTTP-сервер /HTTPPort (тільки Windows, для швидкого старту)

Недокументований параметр платформи, зручний для швидкої перевірки:

```cmd
"C:\Program Files\BAF\8.3.XX.XXXX\bin\1cv8.exe" ENTERPRISE /F "C:\шлях\до\бази" /HTTPPort 8080
```

> Замініть `XX.XXXX` на вашу версію платформи. Вікно 1С відкриється і
> працюватиме як HTTP-сервер — **не закривайте його**, поки сервіс потрібен.

> **Обмеження платформи.** `/HTTPPort` працює **тільки на Windows**. На
> **Linux** використовуйте публікацію через Apache/ibsrv (спосіб 1). На
> **macOS** HTTP-сервіси 1С не підтримуються — піднімайте сервіс на Windows
> (можна у віртуальній машині). Деталі — в [`compatibility.md`](compatibility.md).

#### Перевірка публікації

```sh
# публікація через Apache/IIS (спосіб 1):
curl http://localhost/base/hs/mcp-baf/version

# вбудований HTTP-сервер /HTTPPort (спосіб 2):
curl http://localhost:8080/hs/mcp-baf/version
```

Очікувана відповідь — JSON з версією розширення. Якщо бачите відповідь —
HTTP-сервіс працює, переходьте до кроку 4.

### Крок 4. Налаштувати AI-клієнт

Конфігурація однакова для будь-якого MCP-клієнта і будь-якої моделі:

```json
{
  "mcpServers": {
    "mcp-baf": {
      "command": "/шлях/до/.venv/bin/mcp-baf",
      "args": ["--base", "http://localhost:8080/hs/mcp-baf"],
      "env": {
        "mcp_baf_USER": "Адміністратор",
        "mcp_baf_PASSWORD": "пароль"
      }
    }
  }
}
```

> На Windows шлях зі зворотними слешами подвоюється в JSON:
> `"command": "C:\\шлях\\до\\.venv\\Scripts\\mcp-baf.exe"`.

Щоб увімкнути повнотекстовий пошук по коду (`search_code`), додайте `--dump`
(або env `mcp_baf_DUMP_DIR`) — шлях до вивантаження конфігурації
(`DumpConfigToFiles`):

```json
"args": [
  "--base", "http://localhost:8080/hs/mcp-baf",
  "--dump", "/шлях/до/вивантаження"
]
```

> **Без `--dump` інструмент `search_code` не реєструється в MCP-сервері
> взагалі** — AI-клієнт його навіть не побачить. Індекс будується у фоновому
> потоці: сервер стартує одразу, а `search_code` до завершення індексації
> повертає «index is building, please retry» замість результатів пошуку.

**Claude Code** — однією командою:

```sh
claude mcp add mcp-baf \
  --env mcp_baf_USER='Адміністратор' \
  --env mcp_baf_PASSWORD='пароль' \
  -- /шлях/до/.venv/bin/mcp-baf \
  --base http://localhost:8080/hs/mcp-baf \
  --dump /шлях/до/вивантаження
```

> За замовчуванням сервер додається в поточний проєкт. Щоб зробити його
> доступним усюди, додайте `--scope user`. Перевірити: `claude mcp list`.
> Видалити: `claude mcp remove mcp-baf`.

Повний перелік підтримуваних клієнтів (Claude Desktop, Cursor, Windsurf,
VS Code + Copilot/Continue/Cline, JetBrains IDE, локальні моделі) і їхні
особливості — у [`compatibility.md`](compatibility.md). Усі прапори CLI та
змінні оточення — у [`configuration.md`](configuration.md).

### Крок 5. Перевірити

Перезапустіть AI-клієнт і спитайте:

> Покажи структуру конфігурації моєї бази 1С

AI викличе `get_configuration_info` і покаже ім'я конфігурації, версію,
постачальника, версію платформи. Далі можна досліджувати базу:

> Знайди довідник «Номенклатура» і покажи його реквізити

> Знайди в коді, де обробляється проведення документа «РеалізаціяТоварівПослуг»

Другий запит працює лише якщо запущено з `--dump`.

При старті сервер сам звіряє версію розширення в базі з очікуваною
(`/version`) — розбіжність тихо потрапляє в `server.log` та `audit.log`
(подія `extension_version_mismatch`), не блокуючи роботу. Якщо бачите таке
попередження в логах — перевстановіть розширення: `mcp-baf --install
"шлях/до/бази"`.

## Усунення неполадок

**«1C platform not found» при `--install`.** Платформу 1С не знайдено в
стандартних папках. Вкажіть явно: `--platform "шлях/до/1cv8"`; якщо версію не
видно зі шляху — додайте `--platform-version 8.3.13`.

**«платформа 8.X.Y не поддерживается, минимальная версия 8.3.10» при
`--install`.** Мінімальна підтримувана версія платформи — 8.3.10. На старіших
установка розширення неможлива.

**Помилка про режим сумісності конфігурації при `--install`.** Найчастіша
причина — режим сумісності конфігурації забороняє розширення. Перевірте:
Конфігуратор → Властивості кореня → Режим сумісності — потрібно «Не
використовувати» або «Версія 8.3.11» і новіша. Інші причини: невірні
`--db-user`/`--db-password`, база відкрита в Конфігураторі й заблокована, або
база в режимі лише читання.

**«connection refused» при `curl`.** HTTP-сервіс 1С не запущено. Для
`/HTTPPort` — переконайтеся, що вікно 1С відкрите; для Apache/IIS — що
веб-сервер запущено й публікація активна.

**«404» при зверненні до `/hs/mcp-baf/...`.** Розширення не активне
(Конфігуратор → Розширення → галочка «Активно» напроти `MCP_HTTPService`) або
конфігурацію БД не оновлено (F7 у Конфігураторі).

**AI не бачить інструментів mcp-baf.** Перезапустіть AI-клієнт після зміни
конфігу; перевірте JSON на помилки (коми, неекрановані слеші); переконайтеся,
що шлях до `mcp-baf` повний (абсолютний).

**`search_code` каже «index is building, please retry».** Індекс FTS5
будується у фоновому потоці при першому старті — це нормально для великих
вивантажень. Зачекайте і повторіть запит.

**`search_code` взагалі відсутній у списку інструментів.** Сервер запущено
без `--dump` (і без `mcp_baf_DUMP_DIR`) — без шляху до вивантаження
конфігурації інструмент не реєструється.

**Занадто велика відповідь 1С обривається помилкою.** Збільште
`--max-response-size` (типово 128 MiB) — актуально для великих баз із
розширеннями. Деталі — у [`configuration.md`](configuration.md).
