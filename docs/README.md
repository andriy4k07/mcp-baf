# Документація mcp-baf

Повна документація проєкту. Якщо ви тут уперше — почніть з
[огляду в кореневому README](../README.md), далі переходьте до
[швидкого старту](getting-started.md).

> **Орієнтир.** mcp-baf — **read-only** сторона екосистеми BAF: усі інструменти
> лише читають метадані й дані бази 1С через HTTP-сервіс і нічого в неї не пишуть
> (кожен тул позначений `readOnlyHint`, а `execute_query` пропускає тільки
> `ВЫБРАТЬ`/`SELECT`). Пишуча сторона — сусідній
> [baf-write-mcp](https://github.com/andriy4k07/baf-write-mcp) з потоком
> `propose → validate → create → verify`. Чому саме так — в
> [Архітектурі](architecture.md).

## Зміст

### Початок роботи

- **[Швидкий старт](getting-started.md)** — що це й навіщо, вимоги, повний цикл
  встановлення (пакет, розширення 1С, публікація HTTP-сервісу, AI-клієнт), перша
  перевірка й усунення неполадок.
- **[Архітектура](architecture.md)** — два режими точки входу, патерн реєстрації
  інструментів, наскрізний аудит через `traced_text`, HTTP-клієнт, розширення 1С,
  чому сервер read-only.
- **[Конфігурація](configuration.md)** — усі прапори CLI та змінні оточення,
  пріоритет `defaults → env → CLI`, приклади конфігів клієнта.

### Довідники

- **[Перелік інструментів](tools-reference.md)** — усі 9 інструментів із
  параметрами й обмеженнями, відповідні ендпоінти 1С, плюс 11 промптів.
- **[Пошук по коду та індекс](search-and-index.md)** — SQLite FTS5 над
  вивантаженням, режими `smart`/`regex`/`exact`, BSL-синоніми ru↔en, дисковий
  кеш і NFC-нормалізація на macOS.
- **[Логування та аудит](logging-and-audit.md)** — `audit.log` (JSONL, бізнес-події)
  та `server.log` (операційний), подія `one_c.http`, наскрізний `trace_id`, ротація.
- **[Розробка](development.md)** — тести, структура проєкту, згенерований код,
  правило синхронізації версій розширення, робочий процес.
- **[Сумісність](compatibility.md)** — AI-клієнти, операційні системи, версії
  платформи 1С і Python.

## Що варто знати одразу

| Факт | Деталі |
|---|---|
| Шлях `/hs/mcp-baf` захардкоджений у `RootURL` розширення і мусить збігатися з `--base` | [configuration.md](configuration.md) |
| Без `--dump` / `mcp_baf_DUMP_DIR` інструмент `search_code` **не реєструється взагалі** | [search-and-index.md](search-and-index.md) |
| Індекс будується у фоні — сервер стартує одразу, пошук до готовності просить повторити | [search-and-index.md](search-and-index.md) |
| При зміні `extension_src/` бампати **і** версію розширення, **і** `EXPECTED_EXTENSION_VERSION` | [development.md](development.md) |
| HTTP-сервіси 1С на macOS не підтримуються (сам пакет і `--install` — працюють) | [compatibility.md](compatibility.md) |
| Мінімальна версія платформи 1С — **8.3.10** | [compatibility.md](compatibility.md) |

## Версія

Актуальна версія пакета — `0.1.0` (див. [pyproject.toml](../pyproject.toml)),
версія розширення 1С — `0.4.2` (`server.py:EXPECTED_EXTENSION_VERSION`).
