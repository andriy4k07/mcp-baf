# Сумісність

[← Назад до README](../README.md)

mcp-baf не знає і не визначає, яка модель працює на боці AI-клієнта —
конфігурація одна й та сама для будь-якого MCP-сумісного клієнта.

## AI-клієнти

| Категорія | Клієнти |
|------------|---|
| **Локальні моделі** | Ollama, LM Studio, llama.cpp і будь-які MCP-сумісні клієнти |
| **Хмарні сервіси** | Claude Desktop, Claude Code, GPT (через MCP-клієнт) та інші |
| **IDE** | Cursor, VS Code (GitHub Copilot, Continue, Cline), Windsurf, JetBrains IDE (AI Assistant) |

Детальні інструкції з підключення для конкретного клієнта — в
[`getting-started.md`](getting-started.md#крок-4-налаштувати-ai-клієнт).

> **Кількість інструментів.** mcp-baf реєструє **9 інструментів** (менше без
> `--dump`, коли `search_code` не реєструється) і 11 промптів — набагато
> менше за типові ліміти MCP-клієнтів (наприклад Cursor комфортно тримає
> сумарно до ~40 інструментів по всіх серверах). Обмеження клієнтів на
> кількість MCP-tools для mcp-baf самого по собі проблемою не є.

## Операційні системи

| ОС | MCP-сервер (Python) | `--install` (автоустановка) | HTTP-сервіс 1С |
|----|----------------------|------------------------------|----------------|
| Windows | так | так | так — Apache/IIS (рекомендовано) або `/HTTPPort` (швидкий старт) |
| macOS | так | так | **ні** — HTTP-сервіси 1С на macOS не підтримуються платформою |
| Linux | так | так | так — Apache або ibsrv; `/HTTPPort` недоступний |

> **macOS і HTTP-сервіс 1С.** Сам mcp-baf (сервер і `--install`) працює на
> macOS без обмежень — платформа 1С теж запускається (Конфігуратор,
> `--install` знаходить `1cv8` у `/Applications/1cv8.localized/...`). Але
> **публікація HTTP-сервісів на macOS не підтримується платформою 1С**, тому
> сам сервіс `/hs/mcp-baf`, до якого звертається mcp-baf, треба піднімати на
> Windows або Linux (наприклад у віртуальній машині чи на окремому сервері).
> mcp-baf при цьому може запускатися на macOS і звертатися до сервісу по
> мережі.

> **`/HTTPPort`.** Недокументований параметр запуску платформи 1С, який
> піднімає вбудований HTTP-сервер. Працює **тільки на Windows**. На Linux для
> публікації використовуйте Apache (штатний спосіб) або `ibsrv`
> (вбудований HTTP-сервер 1С для Linux); на macOS альтернативи немає —
> публікація HTTP-сервісів там не підтримується взагалі.

## Версії платформи 1С

| Версія платформи | Статус |
|-------------------|--------|
| до 8.3.10 | Не підтримується — `--install` завершується помилкою |
| 8.3.10 – 8.3.13 | Підтримується. Розширення встановлюється з обхідними патчами (режим сумісності `Version8_3_10`, вирізання елементів XML, невідомих старим платформам) |
| 8.3.14 – 8.3.х (найновіші) | Підтримується без обхідних патчів |
| 8.5.x | Підтримується (окремий формат XML-вивантаження розширення) |

Мінімальна підтримувана версія — **8.3.10**, це жорстке обмеження в
`installer.py`: платформи старші за неї відхиляються ще до звернення до
DESIGNER. Версія платформи визначається автоматично зі шляху до виконуваного
файлу (`--platform`); якщо шлях нестандартний і версії в ньому не видно —
вкажіть її явно через `--platform-version 8.3.13`.

> Розширення підтримує і файлові, і клієнт-серверні (MS SQL, PostgreSQL)
> бази — для клієнт-серверних додайте `--server` і рядок підключення у
> форматі `сервер\база`.

## Python

- **Потрібен Python 3.11+** — і для запуску MCP-сервера, і для режиму
  `--install`.
- Залежності — чистий Python: офіційний MCP SDK, `httpx`, `mcp-baf-audit`
  (спільний пакет аудиту BAF-екосистеми) і стандартна бібліотека (SQLite
  FTS5 для `search_code` — зовнішня СУБД не потрібна).
- Лінтер/форматер у проєкті не налаштовані; для розробки й тестів потрібні
  `dev`-залежності (`pip install -e ".[dev]"`) — деталі в
  [`development.md`](development.md).
